
Тестирование функциональности:
Кейс 1: Создание базы данных с корректными значениями во всех обязательных полях. Проверка, что база данных успешно создана и доступна для использования.
Кейс 2: Создание базы данных без указания обязательных полей. Проверка, что система выдает ошибку и требует заполнить все обязательные поля.

Тестирование валидации данных:
Кейс 1: Попытка создания базы данных с некорректным именем (недопустимые символы, превышение допустимой длины и т.д.). Проверка, что система корректно обрабатывает некорректные значения и выдает соответствующую ошибку.
Кейс 2: Попытка создания базы данных с некорректным значением для поля размера (недопустимый формат, отрицательное значение и т.д.). Проверка, что система производит валидацию значения размера и выдает ошибку при некорректных данных.

Тестирование производительности:
Кейс 1: Создание базы данных с максимально возможными значениями для всех полей (максимальный размер, максимальное количество символов в имени и т.д.). Проверка, что система обрабатывает такие большие значения без задержек и успешно разворачивает базу данных.
Кейс 2: Создание базы данных с разными значениями для поля региона размещения. Измерение времени, требуемого для создания базы данных в разных регионах, и сравнение производительности.


Тестирование безопасности:
Кейс 1: Попытка создания базы данных с использованием SQL-инъекции или других известных уязвимостей. Проверка, что система корректно фильтрует и обрабатывает входные данные и предотвращает возможные атаки.
Кейс 2: Проверка ограничений доступа к созданию базы данных. Попытка создания базы данных без необходимых прав доступа или аутентификации. Проверка, что система требует правильные учетные данные и предоставляет доступ только авторизованным пользователям.
